Projet-DevOps-ImaneBassou

Ce mini-projet a pour objectif de mettre en œuvre les principes fondamentaux du DevOps à travers la mise en place d’une chaîne d’intégration continue (CI). Il utilise un dépôt GitHub, des workflows GitHub Actions et un pipeline Jenkins pour automatiser le cycle de vie d’une application Java simple. L’application sert de support pédagogique pour illustrer l’automatisation du build, l’exécution des tests, la génération d’artefacts et la notification des résultats du pipeline.

Le projet permet de comprendre les principes de l’intégration continue, de mettre en place un dépôt Git structuré, d’automatiser les builds et tests avec GitHub Actions, de configurer un pipeline Jenkins pour la compilation et l’archivage des artefacts, d’assurer la traçabilité des changements via les commits et les Pull Requests, et de mettre en place des notifications du pipeline via Slack.

L’architecture générale repose sur un dépôt GitHub pour la gestion du code source, une application Java basée sur Maven, GitHub Actions pour l’intégration continue, Jenkins pour l’orchestration du pipeline CI, et Slack pour les notifications automatiques. Les technologies utilisées incluent Git et GitHub, GitHub Actions, Jenkins, Java 17, Maven, Docker et Slack. La structure du projet comporte un dossier hello-devops avec les fichiers pom.xml et le code source Java, un dossier .github/workflows contenant le workflow CI, et le fichier README.md.

Le code source est géré via Git et hébergé sur GitHub, avec deux branches principales : main pour la version stable et dev pour le développement et les nouvelles fonctionnalités. Les modifications sont intégrées via des Pull Requests afin de valider le code avant fusion. Le workflow CI défini dans ci.yml récupère automatiquement le code source, configure l’environnement Java 17, compile le projet avec Maven, exécute les tests unitaires et génère le package JAR. Le pipeline Jenkins complète la CI en centralisant l’exécution des builds, incluant les étapes de checkout, build & test, et archivage du JAR généré. Une intégration Slack notifie automatiquement l’état du pipeline après chaque build, indiquant succès ou échec.

Grâce à ce projet, une chaîne d’intégration continue fonctionnelle a été mise en place, les builds et tests sont exécutés automatiquement, les artefacts JAR sont générés et archivés, les notifications sont envoyées automatiquement via Slack et la traçabilité complète des modifications est assurée via GitHub. Des améliorations futures peuvent inclure l’ajout d’un déploiement continu (CD), l’utilisation de Docker Compose pour le déploiement, la mise en place de tests avancés, la sécurisation des accès Jenkins et GitHub, et l’ajout de métriques et monitoring.

Ce projet DevOps a permis d’appliquer concrètement les concepts d’intégration continue avec GitHub Actions et Jenkins, démontrant l’importance de l’automatisation dans le cycle de développement logiciel et constituant une base solide pour des projets DevOps plus avancés.
